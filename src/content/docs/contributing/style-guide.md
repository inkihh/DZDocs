## What this page will cover

- **Voice & tone** — direct, welcoming, never condescending. Concrete starting
  examples of good and bad phrasing.
- **Terminology** — canonical names for engine concepts (P3D, RVMAT, LOD, proxy,
  selection, memory point) so pages stay consistent and searchable.
- **Capitalization & formatting** — headings, code, file paths, key names.
- **Cross-linking conventions** — when and how to link between pages.
- **Images & assets** — sizing, alt text, and the original/vanilla-only rule.
