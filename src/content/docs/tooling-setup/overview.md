Writing scripts and building assets is only half the job — you also need a workflow
to **pack, test, and publish** your mod, and a sane way to organize a project so it
doesn't fight you later.

This section covers the toolchain around the actual content. If you haven't set up
your environment yet, start with [Workbench setup](/getting-started/workbench-setup/).

## What this section covers

- **[Packing](/tooling-setup/packing/)** — turning your source into PBOs the game
  can load.
- **[File patching](/tooling-setup/file-patching/)** — iterating on scripts without
  repacking every change.
- **[Publishing](/tooling-setup/publishing/)** — getting a mod onto the Workshop.
- **[Project workflow](/tooling-setup/project-workflow/)** — directory layout, source
  control, and keeping a project maintainable.

:::note[Related, but separate]
A dedicated **mod project handler** desktop app (create-from-template, packing,
patching, publishing) is a parallel community effort, tracked separately from this
site. These pages document the underlying workflow regardless of which tools you use.
:::
